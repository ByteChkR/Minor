Represents an enumeration for the different affinity domains devices may have.

Enumerator
Numa 

Split the device into sub-devices comprised of compute units that share a NUMA node.

Level4Cache 

Split the device into sub-devices comprised of compute units that share a level 4 data cache.

Level3Cache 

Split the device into sub-devices comprised of compute units that share a level 3 data cache.

Level2Cache 

Split the device into sub-devices comprised of compute units that share a level 2 data cache.

Level1Cache 

Split the device into sub-devices comprised of compute units that share a level 1 data cache.

NextPartitionable 

Split the device along the next partitionable affinity domain. The implementation shall find the first level along which the device or sub-device may be further subdivided in the order NUMA, L4, L3, L2, L1, and partition the device into sub-devices comprised of compute units that share memory subsystems at this level.

enum Engine.OpenCL.DotNetCore.Interop.Devices.DeviceInformation : uint
strong

Represents an enumeration that identifies the device information that can be queried from a device.

Enumerator
Type 

The OpenCL device type.

VendorId 

A unique device vendor identifier. An example of a unique device identifier could be the PCIe ID.

MaximumComputeUnits 

The number of parallel compute units on the OpenCL device. A work-group executes on a single compute unit. The minimum value is 1.

MaximumWorkItemDimensions 

Maximum dimensions that specify the global and local work-item IDs used by the data parallel execution model. (Refer to EnqueueNDRangeKernel). The minimum value is 3 for devices that are not of type DeviceType.Custom.

MaximumWorkGroupSize 

Maximum number of work-items in a work-group that a device is capable of executing on a single compute unit, for any given kernel-instance running on the device. (Refer also to EnqueueNDRangeKernel) and KernelWorkGroupInformation.WorkGroupSize). The minimum value is 1.

MaximumWorkItemSizes 

Maximum number of work-items that can be specified in each dimension of the work-group to EnqueueNDRangeKernel. Retrieves n UIntPtr entries, where n is the value returned by the query for DeviceInformation.MaximumWorkItemDimensions. The minimum value is (1, 1, 1) for devices that are not of type DeviceType.Custom.

PreferredVectorWidthChar 

Preferred native vector width size for built-in scalar types that can be put into vectors. The vector width is defined as the number of scalar elements that can be stored in the vector.

PreferredVectorWidthShort 

Preferred native vector width size for built-in scalar types that can be put into vectors. The vector width is defined as the number of scalar elements that can be stored in the vector.

PreferredVectorWidthInt 

Preferred native vector width size for built-in scalar types that can be put into vectors. The vector width is defined as the number of scalar elements that can be stored in the vector.

PreferredVectorWidthLong 

Preferred native vector width size for built-in scalar types that can be put into vectors. The vector width is defined as the number of scalar elements that can be stored in the vector.

PreferredVectorWidthFloat 

Preferred native vector width size for built-in scalar types that can be put into vectors. The vector width is defined as the number of scalar elements that can be stored in the vector.

PreferredVectorWidthDouble 

Preferred native vector width size for built-in scalar types that can be put into vectors. The vector width is defined as the number of scalar elements that can be stored in the vector. If double precision is not supported, DeviceInformation.PreferredVectorWidthDouble must return 0.

MaximumClockFrequency 

Maximum configured clock frequency of the device in MHz.

AddressBits 

The default compute device address space size of the global address space specified as an unsigned integer value in bits. Currently supported values are 32 or 64 bits.

MaximumReadImageArguments 

Maximum number of image objects arguments of a kernel declared with the read_only qualifier. The minimum value is 128 if DeviceInformation.ImageSupport is true.

MaximumWriteImageArguments 

Maximum number of image objects arguments of a kernel declared with the write_only qualifier. The minimum value is 64 if DeviceInformation.ImageSupport is true. Note: DeviceInformation.MaximumWriteImageArguments is only there for backward compatibility. DeviceInformation.MaximumReadWriteImageArguments should be used instead.

MaximumMemoryAllocationSize 

Maximum size of memory object allocation in bytes. The minimum value is max(min(1024*1024*1024, 1/4th of DeviceInformation.GlobalMemorySize), 32*1024*1024) for devices that are not of type DeviceType.Custom.

Image2DMaximumWidth 

Maximum width of 2D image or 1D image not created from a buffer object in pixels. The minimum value is 16384 if DeviceInformation.ImageSupport is true.

Image2DMaximumHeight 

Maximum height of a 2D image in pixels. The minimum value is 16384 if DeviceInformation.ImageSupport is true.

Image3DMaximumWidth 

Maximum width of 3D image in pixels. The minimum value is 2048 if DeviceInformation.ImageSupport is true.

Image3DMaximumHeight 

Maximum height of 3D image in pixels. The minimum value is 2048 if DeviceInformation.ImageSupport is true.

Image3DMaximumDepth 

Maximum depth of 3D image in pixels. The minimum value is 2048 if DeviceInformation.ImageSupport is true.

ImageSupport 

Is true if images are supported by the OpenCL device and false otherwise.

MaximumParameterSize 

Maximum size in bytes of all arguments that can be passed to a kernel. The minimum value is 1024 for devices that are not of type DeviceType.Custom. For this minimum value, only a maximum of 128 arguments can be passed to a kernel.

MaximumSamplers 

Maximum number of samplers that can be used in a kernel. The minimum value is 16 if DeviceInformation.ImageSupport is true.

MemoryBaseAddressAlignment 

Alignment requirement (in bits) for sub-buffer offsets. The minimum value is the size (in bits) of the largest OpenCL built-in data type supported by the device (long16 in the full profile, long16 or int16 in the embedded profile) for devices that are not of type DeviceType.Custom.

MinimumDataTypeAlignmentSize 

The smallest alignment in bytes which can be used for any data type.

SingleFloatingPointConfiguration 

Describes single precision floating-point capability of the device. For the full profile, the mandated minimum floating-point capability for devices that are not of type DeviceType.Custom is DeviceFloatingPointConfiguration.RoundToNearest | DeviceFloatingPointConfiguration.InfinityAndNotANumber.

GlobalMemoryCacheType 

Type of global memory cache supported. Valid values are: DeviceMemoryCacheType.None, DeviceMemoryCacheType.ReadOnlyCache, and DeviceMemoryCacheType.ReadWriteCache.

GlobalMemoryCachelineSize 

Size of global memory cache line in bytes.

GlobalMemoryCacheSize 

Size of global memory cache in bytes.

GlobalMemorySize 

Size of global device memory in bytes.

MaximumConstantBufferSize 

Maximum size in bytes of a constant buffer allocation. The minimum value is 64 KB for devices that are not of type DeviceType.Custom.

MaximumConstantArguments 

Maximum number of arguments declared with the __constant qualifier in a kernel. The minimum value is 8 for devices that are not of type DeviceType.Custom.

LocalMemoryType 

Type of local memory supported. This can be set to DeviceLocalMemoryType.Local implying dedicated local memory storage such as SRAM, or DeviceLocalMemoryType.Global. For custom devices, DeviceLocalMemoryType.None can also be returned indicating no local memory support.

LocalMemorySize 

Size of local memory region in bytes. The minimum value is 32 KB for devices that are not of type DeviceType.Custom.

ErrorCorrectionSupport 

Is true if the device implements error correction for all accesses to compute device memory (global and constant). Is false if the device does not implement such error correction.

ProfilingTimerResolution 

Describes the resolution of device timer. This is measured in nanoseconds.

EndianLittle 

Is true if the OpenCL device is a little endian device and false otherwise.

Available 

Is true if the device is available and false otherwise. A device is considered to be available if the device can be expected to successfully execute commands enqueued to the .

CompilerAvailable 

Is false if the implementation does not have a compiler available to compile the program source. Is true if the compiler is available. This can be false for the embedded platform profile only.

ExecutionCapabilities 

Describes the execution capabilities of the device. The mandated minimum capability is DeviceExecutionCapability.Kernel.

QueueProperties 

Describes the command-queue properties supported by the device. The mandated minimum capability is CommandQueueProperty.ProfilingEnable.

QueueOnHostProperties 

Describes the on host command-queue properties supported by the device. The mandated minimum capability is CommandQueueProperty.ProfilingEnable.

Name 

Device name string.

Vendor 

Vendor name string.

DriverVersion 

OpenCL software driver version string in the form major_number.minor_number.

Profile 

OpenCL profile string. Retrieves the profile name supported by the device (see note). The profile name returned can be one of the following strings:

FULL_PROFILE - if the device supports the OpenCL specification (functionality defined as part of the core specification and does not require any extensions to be supported).

EMBEDDED_PROFILE - if the device supports the OpenCL embedded profile.

The platform profile returns the profile that is implemented by the OpenCL framework. If the platform profile returned is FULL_PROFILE, the OpenCL framework will support devices that are FULL_PROFILE and may also support devices that are EMBEDDED_PROFILE. The compiler must be available for all devices i.e. DeviceInformation.CompilerAvailable is true. If the platform profile returned is EMBEDDED_PROFILE, then devices that are only EMBEDDED_PROFILE are supported.

Version 

OpenCL version string. Returns the OpenCL version supported by the device. This version string has the following format:

OpenCL[space][major_version.minor_version][space][vendor-specific information]

Extensions 

Retrieves a space separated list of extension names (the extension names themselves do not contain any spaces) supported by the device. The list of extension names returned can be vendor supported extension names and one or more of the following Khronos approved extension names:

cl_khr_int64_base_atomics, cl_khr_int64_extended_atomics, cl_khr_fp16, cl_khr_gl_sharing, cl_khr_gl_event, cl_khr_d3d10_sharing, cl_khr_dx9_media_sharing, cl_khr_d3d11_sharing, cl_khr_gl_depth_images, cl_khr_gl_msaa_sharing, cl_khr_initialize_memory, cl_khr_context_abort, cl_khr_spir, and cl_khr_srgb_image_writes.

The following approved Khronos extension names must be returned by all device that support OpenCL C 2.0:

cl_khr_byte_addressable_store, cl_khr_fp64 (for backward compatibility if double precision is supported), cl_khr_3d_image_writes, cl_khr_image2d_from_buffer, and cl_khr_depth_images.

Please refer to the OpenCL 2.0 Extension Specification for a detailed description of these extensions.

Platform 

The platform associated with this device.

DoubleFloatingPointConfiguration 

Describes double precision floating-point capability of the OpenCL device. Double precision is an optional feature so the mandated minimum double precision floating-point capability is 0. If double precision is supported by the device, then the minimum double precision floating-point capability must be: DeviceFloatingPointConfiguration.FusedMultiplyAdd | DeviceFloatingPointConfiguration.RoundToNearest | DeviceFloatingPointConfiguration.InfinityAndNotANumber | DeviceFloatingPointConfiguration.Denorm.

HalfFloatingPointConfiguration 

Describes the optional half precision floating-point capability of the OpenCL device. The required minimum half precision floating-point capability as implemented by this extension is DeviceFloatingPointConfiguration.RoundToZero or DeviceFloatingPointConfiguration.RoundToInfinity | DeviceFloatingPointConfiguration.InfinityAndNotANumber.

PreferredVectorWidthHalf 

Preferred native vector width size for built-in scalar types that can be put into vectors. The vector width is defined as the number of scalar elements that can be stored in the vector. If the cl_khr_fp16 extension is not supported, DeviceInformation.PreferredVectorWidthHalf must return 0.

HostUnifiedMemory 

Is true if the device and the host have a unified memory subsystem and is false otherwise.

NativeVectorWidthChar 

Retrieves the native ISA vector width. The vector width is defined as the number of scalar elements that can be stored in the vector.

NativeVectorWidthShort 

Retrieves the native ISA vector width. The vector width is defined as the number of scalar elements that can be stored in the vector.

NativeVectorWidthInt 

Retrieves the native ISA vector width. The vector width is defined as the number of scalar elements that can be stored in the vector.

NativeVectorWidthLong 

Retrieves the native ISA vector width. The vector width is defined as the number of scalar elements that can be stored in the vector.

NativeVectorWidthFloat 

Retrieves the native ISA vector width. The vector width is defined as the number of scalar elements that can be stored in the vector.

NativeVectorWidthDouble 

Retrieves the native ISA vector width. The vector width is defined as the number of scalar elements that can be stored in the vector. If double precision is not supported, DeviceInformation.NativeVectorWidthDouble must return 0.

NativeVectorWidthHalf 

Retrieves the native ISA vector width. The vector width is defined as the number of scalar elements that can be stored in the vector. If the cl_khr_fp16 extension is not supported, DeviceInformation.NativeVectorWidthHalf must return 0.

OpenClCVersion 

OpenCL C version string. Retrieves the highest OpenCL C version supported by the compiler for this device that is not of type DeviceType.Custom. This version string has the following format:

OpenCL[space]C[space][major_version.minor_version][space][vendor-specific information] The major_version.minor_version value returned must be 2.0 if DeviceInformation.Version is OpenCL 2.0. The major_version.minor_version value returned must be 1.2 if DeviceInformation.Version is OpenCL 1.2. The major_version.minor_version value returned must be 1.1 if DeviceInformation.Version is OpenCL 1.1. The major_version.minor_version value returned can be 1.0 or 1.1 if DeviceInformation.Version is OpenCL 1.0.

LinkerAvailable 

Is false if the implementation does not have a linker available. Is true if the linker is available. This can be false for the embedded platform profile only. This must be true if DeviceInformation.CompilerAvailable is true.

BuiltInKernels 

A semi-colon separated list of built-in kernels supported by the device. An empty string is returned if no built-in kernels are supported by the device.

ImageMaximumBufferSize 

Maximum number of pixels for a 1D image created from a buffer object. The minimum value is 65536 if DeviceInformation.ImageSupport is true.

ImageMaximumArraySize 

Maximum number of images in a 1D or 2D image array. The minimum value is 2048 if DeviceInformation.ImageSupport is true.

ParentDevice 

Retrieves the handle of the parent device to which this sub-device belongs. If device is a root-level device, a null value is returned.

PartitionMaximumSubDevices 

Retrieves the maximum number of sub-devices that can be created when a device is partitioned. The value returned cannot exceed DeviceInformation.MaximumComputeUnits.

PartitionProperties 

Retrieves the list of partition types supported by device. If the device cannot be partitioned (i.e. there is no partitioning scheme supported by the device that will return at least two subdevices), a value of 0 will be returned.

PartitionAffinityDomain 

Retrieves the list of supported affinity domains for partitioning the device using DevicePartitionProperty.PartitionByAffinityDomain. If the device does not support any affinity domains, a value of 0 will be returned.

PartitionType 

Retrieves the properties argument specified in CreateSubDevices if the device is a sub-device. In the case where the properties argument to CreateSubDevices is DevicePartitionProperty.PartitionByAffinityDomain, DeviceAffinityDomain.NextPartitionable, the affinity domain used to perform the partition will be returned. Otherwise the implementation may either return a parameterValueSize of 0 i.e. there is no partition type associated with device or can return a property value of 0 (where 0 is used to terminate the partition property list) in the memory that parameterValue points to.

ReferenceCount 

Retrieves the device reference count. If the device is a root-level device, a reference count of one is returned.

PreferredInteropUserSync 

Is true if the device's preference is for the user to be responsible for synchronization, when sharing memory objects between OpenCL and other APIs such as DirectX, false if the device/implementation has a performant path for performing synchronization of memory object shared between OpenCL and other APIs such as DirectX.

PrintfBufferSize 

Maximum size in bytes of the internal buffer that holds the output of printf calls from a kernel. The minimum value for the full profile is 1 MB.

ImagePitchAlignment 

The row pitch alignment size in pixels for 2D images created from a buffer. The value returned must be a power of 2. If the device does not support images, this value must be 0.

ImageBaseAddressAlignment 

This query should be used when a 2D image is created from a buffer which was created using MemoryFlag.UseHostPointer. The value returned must be a power of 2. This query specifies the minimum alignment in pixels of the hostPointer specified to CreateBuffer. If the device does not support images, this value must be 0.

MaximumReadWriteImageArguments 

Maximum number of image objects arguments of a kernel declared with the write_only or read_write qualifier. The minimum value is 64 if DeviceInformation.ImageSupport is true. Note: DeviceInformation.MaximumWriteImageArguments is only there for backward compatibility. DeviceInformation.MaximumReadWriteImageArguments should be used instead.

MaximumGlobalVariableSize 

The maximum number of bytes of storage that may be allocated for any single variable in program scope or inside a function in OpenCL C declared in the global address space. The minimum value is 64 KB.

QueueOnDeviceProperties 

Describes the on device command-queue properties supported by the device. The mandated minimum capability is CommandQueueProperty.OutOfOrderExecutionModeEnable | CommandQueueProperty.ProfilingEnable.

QueueOnDevicePreferredSize 

The size of the device queue in bytes preferred by the implementation. Applications should use this size for the device queue to ensure good performance. The minimum value is 16 KB.

QueueOnDeviceMaximumSize 

The maximum size of the device queue in bytes. The minimum value is 256 KB for the full profile and 64 KB for the embedded profile.

MaximumOnDeviceQueues 

The maximum number of device queues that can be created per context. The minimum value is 1.

MaximumOnDeviceEvents 

The maximum number of events in use by a device queue. These refer to events returned by the enqueue_ built-in functions to a device queue or user events returned by the create_user_event built-in function that have not been released. The minimum value is 1024.

SvmCapabilities 

Describes the various shared virtual memory (a.k.a. SVM) memory allocation types the device supports. Coarse-grain SVM allocations are required to be supported by all OpenCL 2.0 devices. The mandated minimum capability is DeviceSvmCapabilities.CoarseGrainBuffer.

GlobalVariablePreferredTotalSize 

Maximum preferred total size, in bytes, of all program variables in the global address space. This is a performance hint. An implementation may place such variables in storage with optimized device access. This query returns the capacity of such storage. The minimum value is 0.

MaximumPipeArguments 

The maximum number of pipe objects that can be passed as arguments to a kernel. The minimum value is 16.

PipeMaximumActiveReservations 

The maximum number of reservations that can be active for a pipe per work-item in a kernel. A work-group reservation is counted as one reservation per work-item. The minimum value is 1.

PipeMaximumPacketSize 

The maximum size of pipe packet in bytes. The minimum value is 1024 bytes.

PreferredPlatformAtomicAlignment 

Retrieves the value representing the preferred alignment in bytes for OpenCL 2.0 fine-grained SVM atomic types. This query can return 0 which indicates that the preferred alignment is aligned to the natural size of the type.

PreferredGlobalAtomicAlignment 

Retrieves the value representing the preferred alignment in bytes for OpenCL 2.0 atomic types to global memory. This query can return 0 which indicates that the preferred alignment is aligned to the natural size of the type.

PreferredLocalAtomicAlignment 

Retrieves the value representing the preferred alignment in bytes for OpenCL 2.0 atomic types to local memory. This query can return 0 which indicates that the preferred alignment is aligned to the natural size of the type.

IntermediateLanguageVersion 

The intermediate languages that can be supported by CreateProgramWithIL for this device. Retrieves a space-separated list of IL version strings of the form <IL_Prefix>_<Major_Version>.<Minor_Version>. For OpenCL 2.1, "SPIR-V" is a required IL prefix.

MaximumNumberOfSubGroups 

Maximum number of sub-groups in a work-group that a device is capable of executing on a single compute unit, for any given kernel-instance running on the device. The minimum value is 1. (Refer also to GetKernelSubGroupInfo.)

SubGroupIndependentForwardProgress 

Is true if this device supports independent forward progress of sub-groups, false otherwise. If cl_khr_subgroups is supported by the device this must return true.

TerminateCapabilityKhr 

If the cl_khr_terminate_context extension is enabled, describes the termination capability of the OpenCL device. This is a bitfield where a value of TerminateCapabilityContextKhr indicates that context termination is supported.

SpirVersion 

If the cl_khr_spir extension is enabled, a space separated list of SPIR versions supported by the device. For example returning “1.2 2.0” in this query implies that SPIR version 1.2 and 2.0 are supported by the implementation.

strong

Represents an enumration for the different SVM capabilities a device may have.

Enumerator
CoarseGrainBuffer 

Support for coarse-grain buffer sharing using SvmAllocate. Memory consistency is guaranteed at synchronization points and the host must use calls to EnqueueMapBuffer and EnqueueUnmapMemoryObject.

FineGrainBuffer 

Support for fine-grain buffer sharing using SvmAllocate. Memory consistency is guaranteed at synchronization points without need for EnqueueMapBuffer and EnqueueUnmapMemoryObject.

FineGrainSystem 

Support for sharing the host’s entire virtual memory including memory allocated using malloc. Memory consistency is guaranteed at synchronization points.

Atomics 

Support for the OpenCL 2.0 atomic operations that provide memory consistency across the host and all OpenCL devices supporting fine-grain SVM allocations.

◆ DeviceType

enum Engine.OpenCL.DotNetCore.Interop.Devices.DeviceType : ulong
strong

Represents the different device types that can be supported by an OpenCL platform.

Enumerator
Default 

The default OpenCL device in the system. The default device cannot be a DeviceType.Custom device.

Cpu 

An OpenCL device that is the host processor. The host processor runs the OpenCL implementations and is a single or multi-core CPU.

Gpu 

An OpenCL device that is a GPU. By this we mean that the device can also be used to accelerate a 3D API such as OpenGL or DirectX.

Accelerator 

Dedicated OpenCL accelerators (for example the IBM CELL Blade). These devices communicate with the host processor using a peripheral interconnect such as PCIe.

Custom 

Dedicated accelerators that do not support programs written in OpenCL C.

All 

All OpenCL devices available in the system except DeviceType.Custom devices.
