static Result Engine.OpenCL.DotNetCore.Interop.Kernels.KernelsNativeApi.SetKernelArgument ( [In] IntPtr  kernel,
[MarshalAs(UnmanagedType.U4)] uint  argumentIndex,
[In] UIntPtr  argumentSize,
[In] IntPtr  argumentValue 
)

Set the argument value for a specific argument of a kernel.

Parameters
kernelA valid kernel object.
argumentIndexThe argument index. Arguments to the kernel are referred by indices that go from 0 for the leftmost argument to n - 1, where n is the total number of arguments declared by a kernel.
argumentSizeSpecifies the size of the argument value. If the argument is a memory object, the size is the size of the memory object. For arguments declared with the local qualifier, the size specified will be the size in bytes of the buffer that must be allocated for the local argument. If the argument is of type sampler_t, the argumentSize value must be equal to sizeof(cl_sampler). If the argument is of type queue_t, the argumentSize value must be equal to sizeof(cl_commandQueue). For all other arguments, the size will be the size of argument type.
argumentValueA pointer to data that should be used as the argument value for argument specified by argumentIndex. The argument data pointed to by argumentValue is copied and the argumentValue pointer can therefore be reused by the application after SetKernelArgument returns. The argument value specified is the value used by all API calls that enqueue kernel (EnqueueNDRangeKernel) until the argument value is changed by a call to SetKernelArgument for kernel.
Returns
Returns Result.Success if the function is executed successfully. Otherwise, it returns an error.

◆ SetKernelArgumentSvmPointer()

static Result Engine.OpenCL.DotNetCore.Interop.Kernels.KernelsNativeApi.SetKernelArgumentSvmPointer ( [In] IntPtr  kernel,
[MarshalAs(UnmanagedType.U4)] uint  argumentIndex,
[In] IntPtr  argumentValue 
)

Set a SVM pointer as the argument value for a specific argument of a kernel.

Parameters
kernelA valid kernel object.
argumentIndexThe argument index. Arguments to the kernel are referred by indices that go from 0 for the leftmost argument to n - 1, where n is the total number of arguments declared by a kernel.
argumentValueThe SVM pointer that should be used as the argument value for argument specified by argumentIndex. The SVM pointer specified is the value used by all API calls that enqueue kernel (EnqueueNDRangeKernel) until the argument value is changed by a call to SetKernelArgumentSvmPointer for kernel. The SVM pointer can only be used for arguments that are declared to be a pointer to global or constant memory. The SVM pointer value must be aligned according to the argument type. For example, if the argument is declared to be global float4 *p, the SVM pointer value passed for p must be at a minimum aligned to a float4. The SVM pointer value specified as the argument value can be the pointer returned by SvmAllocate or can be a pointer + offset into the SVM region.
Returns

Returns Result.Success if the function is executed successfully. Otherwise, it returns one of the following errors:

Result.InvalidKernel if kernel is not a valid kernel object.

Result.InvalidArgumentIndex if argumentIndex is not a valid argument index.

Result.InvalidArgumentValue if argumentValue is not a valid value.

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.

Result.OutOfHostMemory if there is a failure to allocate resources required by the OpenCL implementation on the host.
