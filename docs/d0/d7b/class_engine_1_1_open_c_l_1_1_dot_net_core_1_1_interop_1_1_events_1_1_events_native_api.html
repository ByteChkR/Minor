◆ ReleaseEvent()

static Result Engine.OpenCL.DotNetCore.Interop.Events.EventsNativeApi.ReleaseEvent ( [In] IntPtr  eventPointer)

Decrements the event reference count. The event object is deleted once the reference count becomes zero, the specific command identified by this event has completed (or terminated) and there are no commands in the command-queues of a context that require a wait for this event to complete. Using this function to release a reference that was not obtained by creating the object or by calling RetainEvent causes undefined behavior. Developers should be careful when releasing their last reference count on events created by CreateUserEvent that have not yet been set to status of CommandExecutionStatus.Complete or an error. If the user event was used in the eventWaitList argument passed to a Enqueue*** API or another application host thread is waiting for it in WaitForEvents, those commands and host threads will continue to wait for the event status to reach CommandExecutionStatus.Complete or error, even after the user has released the object. Since in this scenario the developer has released his last reference count to the user event, it would be in principle no longer valid for him to change the status of the event to unblock all the other machinery. As a result the waiting tasks will wait forever, and associated events, memory objects, command queues and contexts are likely to leak. In-order command queues caught up in this deadlock may cease to do any work.

Parameters
eventPointerEvent object being released.
Returns

Returns Result.Success if the function is executed successfully. Otherwise, it returns the following:

Result.InvalidEvent if event is not a valid event object.

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.

Result.OutOfHostMemory if there is a failure to allocate resources required by the OpenCL implementation on the host.

Registers a user callback function for a specific command execution status.

Parameters
eventPointerA valid event object.
commandExecutionCallbackTypeSpecifies the command execution status for which the callback is registered. The command execution callback values for which a callback can be registered are CommandExecutionStatus.Submitted, CommandExecutionStatus.Running, or CommandExecutionStatus.Complete. There is no guarantee that the callback functions registered for various execution status values for an event will be called in the exact order that the execution status of a command changes. Furthermore, it should be noted that receiving a call back for an event with a status other than CommandExecutionStatus.Complete, in no way implies that the memory model or execution model as defined by the OpenCL specification has changed. For example, it is not valid to assume that a corresponding memory transfer has completed unless the event is in a state CommandExecutionStatus.Complete. The callback function registered for a commandExecutionCallbackType value of CommandExecutionStatus.Complete will be called when the command has completed successfully or is abnormally terminated.
notificationCallbackThe event callback function that can be registered by the application. This callback function may be called asynchronously by the OpenCL implementation. It is the application's responsibility to ensure that the callback function is thread-safe. The parameters to this callback function are:

event is the event object for which the callback function is invoked.

eventCommandExecutionStatus represents the execution status of command for which this callback function is invoked. If the callback is called as the result of the command associated with event being abnormally terminated, an appropriate error code for the error that caused the termination will be passed to eventCommandExecutionStatus instead.

userData is a pointer to user supplied data.

Parameters
userDataWill be passed as the userData argument when notificationCallback is called. userData can be null.
Returns

Returns Result.Success if the function is executed successfully. Otherwise, it returns the following:

Result.InvalidEvent if event is not a valid event object.

Result.InvalidValue if notificationCallback is null or if commandExecutionCallbackType is not CommandExecutionStatus.Complete.

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.

Result.OutOfHostMemory if there is a failure to allocate resources required by the OpenCL implementation on the host.

[MarshalAs(UnmanagedType.I4)] int  executionStatus

Sets the execution status of a user event object. If there are enqueued commands with user events in the eventWaitList argument of Enqueue*** commands, the user must ensure that the status of these user events being waited on are set using SetUserEventStatus before any OpenCL APIs that release OpenCL objects except for event objects are called, otherwise the behavior is undefined.

Parameters
eventPointerA user event object created using CreateUserEvent.
executionStatusSpecifies the new execution status to be set and can be CommandExecutionStatus.Complete or a negative integer value to indicate an error. A negative integer value causes all enqueued commands that wait on this user event to be terminated. SetUserEventStatus can only be called once to change the execution status of event.
Returns

Returns Result.Success if the function is executed successfully. Otherwise, it returns the following:

Result.InvalidEvent if event is not a valid user event object.

Result.InvalidValue if executionStatus is not CommandExecutionStatus.Complete or a negative integer value.

Result.InvalidOperation if the executionStatus for event has already been changed by a previous call to SetUserEventStatus.

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.

Result.OutOfHostMemory if there is a failure to allocate resources required by the OpenCL implementation on the host.

Waits on the host thread for commands identified by event objects in eventList to complete. A command is considered complete if its execution status is CommandExecutionState.Complete or a negative value. The events specified in eventList act as synchronization points.

Parameters
numberOfEventsThe number of event contained in eventList.
eventListA list of event on which is to be waited on the host thread.
Returns

Returns Result.Success if the execution status of all events in eventList is CommandExecutionState.Complete. Otherwise, it returns one of the following errors:

Result.InvalidValue if numberOfEvents is zero or eventList is null.

Result.InvalidContext if events specified in eventList do not belong to the same context.

Result.InvalidEvent if event objects specified in eventList are not valid event objects.

Result.ExecutionStatusErrorForEventsInWaitList if the execution status of any of the events in eventList is a negative integer value.

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.

Result.OutOfHostMemory if there is a failure to allocate resources required by the OpenCL implementation on the host.
