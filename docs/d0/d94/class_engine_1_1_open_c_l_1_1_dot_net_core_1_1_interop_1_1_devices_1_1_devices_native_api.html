[MarshalAs(UnmanagedType.U4)] out uint  numberOfDevicesReturned 
)

Creates an array of sub-devices that each reference a non-intersecting set of compute units within inDevice.

Parameters
inDeviceThe device to be partitioned.
propertiesSpecifies how inDevice is to be partitioned, described by a partition name and its corresponding value. Each partition name is immediately followed by the corresponding desired value. The list is terminated with 0.
numberOfDevicesSize of memory pointed to by outDevices specified as the number of device entries.
outDevices
numberOfDevicesReturnedThe buffer where the OpenCL sub-devices will be returned. If outDevices is null, this argument is ignored. If outDevices is not null, numberOfDevices must be greater than or equal to the number of sub-devices that device may be partitioned into according to the partitioning scheme specified in properties.
Returns

Returns Result.Success if the function is executed successfully. Otherwise, it returns the following:

Result.InvalidDevice if device is not valid.

Result.InvalidValue if values specified in properties are not valid or if values specified in properties are valid but not supported by the device. If outDevices is not null and numberOfDevices is less than the number of sub-devices created by the partition scheme.

Result.DevicePartitionFailed if the partition name is supported by the implementation but inDevice could not be further partitioned.

Result.InvalidDevicePartitionCount if the partition name specified in properties is DevicePartitionProperty.PartitionByCounts and the number of sub-devices requested exceeds the maximum number of sub-devices or the total number of compute units requested exceeds the maximum number of compute units for inDevice, or the number of compute units requested for one or more sub-devices is less than zero or the number of sub-devices requested exceeds the maximum number of compute units for inDevice.

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.

Result.OutOfHostMemory if there is a failure to allocate resources required by the OpenCL implementation on the host.

[In] IntPtr  hostTimestamp 
)

Returns a reasonably synchronized pair of timestamps from the device timer and the host timer as seen by device.

Parameters
deviceA device returned by GetDeviceIds.
deviceTimestampWill be updated with the value of the current timer in nanoseconds. The resolution of the timer is the same as the device profiling timer returned by GetDeviceInformation and the DeviceInformation.ProfilingTimerResolution query.
hostTimestampWill be updated with the value of the current timer in nanoseconds at the closest possible point in time to that at which device timer was returned. The resolution of the timer may be queried via GetPlatformInfo and the flag PlatformInformation.HostTimerResolution.
Returns

Returns Result.Success with a time value in hostTimestamp if provided. Otherwise, it returns one of the following errors:

Result.InvalidDevice if device is not a valid OpenCL device.

Result.InvalidValue if hostTimestamp or deviceTimestamp is null.

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.

Result.OutOfHostMemory if there is a failure to allocate resources required by the OpenCL implementation on the host.

static Result Engine.OpenCL.DotNetCore.Interop.Devices.DevicesNativeApi.GetDeviceIds ( [In] IntPtr  platform,

)

Obtain the list of devices available on a platform.

Parameters
platformRefers to the platform ID returned by <see cref="GetPlatformIds"/< or can be null. If platform is null, the behavior is implementation-defined.
deviceTypeA bitfield that identifies the type of OpenCL device. The deviceType can be used to query specific OpenCL devices or all OpenCL devices available.
numberOfEntriesThe number of device entries that can be added to devices. If devices is not null, the numberOfEntries must be greater than zero.
devicesA list of OpenCL devices found. The device values returned in devices can be used to identify a specific OpenCL device. If devices argument is null, this argument is ignored. The number of OpenCL devices returned is the mininum of the value specified by numberOfEntries or the number of OpenCL devices whose type matches deviceType.
numberOfDevicesReturnedThe number of OpenCL devices available that match <see cref="deviceType". If numberOfDevicesReturned is null, this argument is ignored.
Returns

Returns Result.Success if the function is executed successfully. Otherwise it returns one of the following errors:

Result.InvalidPlatform if platform is not a valid platform.

Result.InvalidDeviceType if deviceType is not a valid value.

Result.InvalidValue if numberOfEntries is equal to zero and devices is not null or if both numberOfDevicesReturned and devices are null.

Result.DeviceNotFound if no OpenCL devices that matched deviceType were found.

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.

Result.OutOfHostMemory if there is a failure to allocate resources required by the OpenCL implementation on the host.

parameterNameAn enumeration constant that identifies the device information being queried.

parameterValueA pointer to memory location where appropriate values for a given parameterName. If parameterValue is null, it is ignored.
parameterValueSizeReturnedReturns the actual size in bytes of data being queried by parameterValue. If parameterValueSizeReturned is null, it is ignored.
Returns

Result.OutOfResources if there is a failure to allocate resources required by the OpenCL implementation on the device.
