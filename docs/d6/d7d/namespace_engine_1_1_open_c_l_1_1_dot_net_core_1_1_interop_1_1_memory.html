Only an alpha channel.

Rgba 

Red, green, blue, and alpha channel.

Blue, green, red, and alpha channel. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8, ChannelType.NormalizedSignedInteger8, ChannelType.SignedInteger8, or ChannelType.UnsignedInteger8.

Argb 

Alpha, red, green, and blue channel. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8, ChannelType.NormalizedSignedInteger8, ChannelType.SignedInteger8, or ChannelType.UnsignedInteger8.

Intensity 

Only one channel for the intensity. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8, ChannelType.NormalizedUnsignedInteger16, ChannelType.NormalizedSignedInteger8, ChannelType.NormalizedSignedInteger16, ChannelType.HalfFloat, or ChannelType.Float.

Luminance 

Only one channel for the luminance. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8, ChannelType.NormalizedUnsignedInteger16, ChannelType.NormalizedSignedInteger8, ChannelType.NormalizedSignedInteger16, ChannelType.HalfFloat, or ChannelType.Float.

Rx 

Only a red channel with some padding.

Rgx 

Red and green channel with some padding.

Rgbx 

Red, green, and blue channel with some padding. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedShortFloat565, ChannelType.NormalizedUnsignedShortFloat555 or ChannelType.NormalizedUnsignedInteger101010.

Depth 

Only one channel for depth. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger16 or ChannelType.Float.

DepthStencil 

Only one channel for a depth stencil. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger24.

Srgb 

Standard RGB (red, green, and blue channel). This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8.

Srgbx 

Standard RGB (red, green, and blue channel) with some padding. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8.

Srgba 

Standard RGBA with alpha channel (red, green, blue, and alpha channel). This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8.

Sbgra 

Standard BGRA with alpha channel (blue, green, red, and alpha channel). This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8.

Abgr 

Alpha, blue, green, and red channel. This channel order can only be used if the channel data type is ChannelType.NormalizedUnsignedInteger8, ChannelType.NormalizedSignedInteger8, ChannelType.SignedInteger8, or ChannelType.UnsignedInteger8.

enum Engine.OpenCL.DotNetCore.Interop.Memory.MemoryFlag : ulong
strong

Represents the memory flags, that are used to create memory buffers.

Enumerator
ReadWrite 

This flag specifies that the memory object will be read and written by a kernel. This is the default.

WriteOnly 

This flags specifies that the memory object will be written but not read by a kernel. Reading from a buffer or image object created with WriteOnly inside a kernel is undefined. ReadWrite and WriteOnly are mutually exclusive.

ReadOnly 

This flag specifies that the memory object is a read-only memory object when used inside a kernel. Writing to a buffer or image object created with ReadOnly inside a kernel is undefined. ReadWrite or WriteOnly and ReadOnly are mutually exclusive.

UseHostPointer 

This flag is valid only if a host pointer was specified. If specified, it indicates that the application wants the OpenCL implementation to use memory referenced by the specified host pointer as the storage bits for the memory object. OpenCL implementations are allowed to cache the buffer contents pointed to by the specified host pointer in device memory. This cached copy can be used when kernels are executed on a device. The result of OpenCL commands that operate on multiple buffer objects created with the same host pointer or overlapping host regions is considered to be undefined. Refer to the description of the alignment rules for the specified host pointer for memory objects (buffer and images) created using UseHostPointer.

AllocateHostPointer 

This flag specifies that the application wants the OpenCL implementation to allocate memory from host accessible memory. AllocateHostPointer and UseHostPointer are mutually exclusive.

CopyHostPointer 

This flag is valid only if a host pointer was specified. If specified, it indicates that the application wants the OpenCL implementation to allocate memory for the memory object and copy the data from memory referenced by the specified host pointer. CopyHostPointer and UseHostPointer are mutually exclusive. CopyHostPointer can be used with AllocateHostPointer to initialize the contents of the memory object allocated using host-accessible (e.g. PCIe) memory.

HostWriteOnly 

This flag specifies that the host will only write to the memory object (using OpenCL APIs that enqueue a write or a map for write). This can be used to optimize write access from the host (e.g. enable write combined allocations for memory objects for devices that communicate with the host over a system bus such as PCIe).

HostReadOnly 

This flag specifies that the host will only read the memory object (using OpenCL APIs that enqueue a read or a map for read). HostWriteOnly and HostReadOnly are mutually exclusive.

HostNoAccess 

This flag specifies that the host will not read or write the memory object. HostWriteOnly or HostReadOnly and HostNoAccess are mutually exclusive.

KernelReadAndWrite 

Can be used to get a list of supported image formats that can be both read from and written to by a kernel.

strong

Represents an enumeration for the different types of information that can be queried from an memory object.

Enumerator
Type 

The type of the memory object.

Flags 

The flags argument value specified when the memory object was created.

Size 

The actual size of the data store associated with memory object in bytes.

HostPointer 

The host pointer argument value specified when memory object is created.

MapCount 

The map count. The map count returned should be considered immediately stale. It is unsuitable for general use in applications. This feature is provided for debugging.

ReferenceCount 

The reference count. The reference count returned should be considered immediately stale. It is unsuitable for general use in applications. This feature is provided for identifying memory leaks.

Context 

The context specified when the memory object was created.

AssociatedMemoryObject 

The memory object from which the memory object was created.

Offset 

The offset if memory object is a sub-buffer object created using CreateSubBuffer.

UsesSvmPointer 

true if the memory object is a buffer object that was created with MemoryFlag.UseHostPointer or is a subbuffer object of a buffer object that was created with MemoryFlag.UseHostPointer and the host pointer specified when the buffer object was created is a SVM pointer. Otherwise false.
