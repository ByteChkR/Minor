enum Engine.OpenCL.DotNetCore.Interop.CommandQueues.CommandQueueProperty : ulong
strong

Represents an enumeration for the command queue properties.

Enumerator
OutOfOrderExecutionModeEnable 

Determines whether the commands queued in the command-queue are executed in-order or out-of-order. If set, the commands in the command-queue are executed out-of-order. Otherwise, commands are executed in-order.

ProfilingEnable 

Enables or disables profiling of commands in the command-queue. If set, the profiling of commands is enabled. Otherwise profiling of commands is disabled.

OnDevice 

Indicates that the command-queue is a device queue. If CommandQueueProperty.OnDevice is set, CommandQueuePropertyOutOfOrderExecutionModeEnable must also be set. Only out-of-order device queues are supported.

OnDeviceDefault 

Indicates that this is the default device queue. This can only be used with CommandQueueProperty.OnDevice. The application must create the default device queue if any kernels containing calls to GetDefaultQueue are enqueued. There can only be one default device queue for each device within a context. CreateCommandQueueWithProperties with CommandQueueInformation.Properties set to CommandQueueProperty.OnDevice | CommandQueueProperty.OnDeviceDefault will return the default device queue that has already been created and increment its retain count by 1.
