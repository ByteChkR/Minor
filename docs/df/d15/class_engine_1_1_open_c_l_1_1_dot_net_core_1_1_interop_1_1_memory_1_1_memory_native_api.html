)

Creates a buffer object.

Parameters
contextA valid OpenCL context used to create the buffer object.
flagsAn enumeration that is used to specify allocation and usage information such as the memory arena that should be used to allocate the buffer object and how it will be used. If value specified for flags is 0, the default is used which is MemoryFlag.ReadWrite.
sizeThe size in bytes of the buffer memory object to be allocated.
hostPointerA pointer to the buffer data that may already be allocated by the application. The size of the buffer that hostPointer points to must be greater or equal than size bytes.
errorCodeReturns an appropriate error code. If errorCode is null, no error code is returned.
Returns
Returns a valid non-zero buffer object and errorCode is set to Result.Success if the buffer object is created successfully. Otherwise, it returns a null value and an error value in errorCode.

Creates a 2D image object.

Parameters
contextA valid OpenCL context on which the image object is to be created.
flagsAn enumeration that is used to specify allocation and usage information about the image memory object being created.
imageFormatA pointer to a structure that describes format properties of the image to be allocated.
imageWidthThe widthof the image in pixels. This must be values greater than or equal to 1.
imageHeightThe height of the image in pixels. This must be values greater than or equal to 1.
imageRowPitchThe scan-line pitch in bytes. This must be 0 if hostPointer is null and can be either 0 or greater than or equal to imageWidth * size of element in bytes if hostPointer is not null. If hostPointer is not null and imageRowPitch is equal to 0, imageRowPitch is calculated as imageWidth * size of element in bytes. If imageRowPitch is not 0, it must be a multiple of the image element size in bytes.
hostPointerA pointer to the image data that may already be allocated by the application. The size of the buffer that hostPointer points to must be greater than or equal to imageRowPitch * imageHeight. The size of each element in bytes must be a power of 2. The image data specified by hostPointer is stored as a linear sequence of adjacent scanlines. Each scanline is stored as a linear sequence of image elements.
errorCodeWill return an appropriate error code. If errorCode is null, no error code is returned.
Returns
Returns a valid non-zero image object and errorCode is set to Result.Success if the image object is created successfully. Otherwise, it returns a null value with an error value returned in errorCode.

Creates a 3D image object.

Parameters
contextA valid OpenCL context on which the image object is to be created.
flagsAn enumeration that is used to specify allocation and usage information about the image memory object being created.
imageFormatA pointer to a structure that describes format properties of the image to be allocated.
imageWidthThe widthof the image in pixels. This must be values greater than or equal to 1.
imageHeightThe height of the image in pixels. This must be values greater than or equal to 1.
imageDepthThe depth of the image in pixels. This must be values greater than or equal to 1.
imageRowPitchThe scan-line pitch in bytes. This must be 0 if hostPointer is null and can be either 0 or greater than or equal to imageWidth * size of element in bytes if hostPointer is not null. If hostPointer is not null and imageRowPitch is equal to 0, imageRowPitch is calculated as imageWidth * size of element in bytes. If imageRowPitch is not 0, it must be a multiple of the image element size in bytes.
imageSlicePitchThe size in bytes of each 2D slice in the 3D image. This must be 0 if hostPointer is null and can be either 0 or greater than or equal to imageRowPitch * imageHeight if hostPointer is not null. If hostPointer is not null and imageSlicePitch equal to 0, imageSlicePitch is calculated as imageRowPitch * imageHeight. If imageSlicePitch is not 0, it must be a multiple of the imageRowPitch.
hostPointerA pointer to the image data that may already be allocated by the application. The size of the buffer that hostPointer points to must be greater than or equal to imageSlicePitch * imageDepth. The size of each element in bytes must be a power of 2. The image data specified by hostPointer is stored as a linear sequence of adjacent 2D slices. Each 2D slice is a linear sequence of adjacent scanlines. Each scanline is a linear sequence of image elements.
errorCodeWill return an appropriate error code. If errorCode is null, no error code is returned.
Returns
Returns a valid non-zero image object and errorCode is set to Result.Success if the image object is created successfully. Otherwise, it returns a null value with an error value returned in errorCode.

Creates a pipe object.

Parameters
contextA valid OpenCL context used to create the pipe object.
flagsAn enumeration that is used to specify allocation and usage information such as the memory arena that should be used to allocate the pipe object and how it will be used. Only MemoryFlag.ReadOnly, MemoryFlag.WriteOnly, MemoryFlag.ReadWrite, and MemoryFlag.HostNoAccess can be specified when creating a pipe object. If value specified for flags is 0, the default is used which is MemoryFlag.ReadWrite | MemoryFlag.HostNoAccess.
pipePacketSizeSize in bytes of a pipe packet.
pipeMaximumNumberOfPacketsSpecifies the pipe capacity by specifying the maximum number of packets the pipe can hold.
propertiesA list of properties for the pipe and their corresponding values. Each property name is immediately followed by the corresponding desired value. The list is terminated with 0. In OpenCL 2.0, properties must be null.
errorCodeReturns an appropriate error code. If errorCode is null, no error code is returned.
Returns
Returns a valid non-zero pipe object and errorCode is set to Result.Success if the pipe object is created successfully. Otherwise, it returns a null value with an error value returned in errorCode.

Creates a new buffer object (referred to as a sub-buffer object) from an existing buffer object.

Parameters
memoryObjectA valid buffer object and cannot be a sub-buffer object.
flagsAn enumeration that is used to specify allocation and usage information about the sub-buffer memory object being created and is described in the table below. If the MemoryFlag.ReadWrite, MemoryFlag.ReadOnly or MemoryFlag.WriteOnly values are not specified in flags, they are inherited from the corresponding memory access qualifers associated with buffer. The MemoryFlag.UseHostPointer, MemoryFlag.AllocateHostPointer and MemoryFlag.CopyHostPointer values cannot be specified in flags but are inherited from the corresponding memory access qualifiers associated with buffer. If MemoryFlag.CopyHostPointer is specified in the memory access qualifier values associated with buffer it does not imply any additional copies when the sub-buffer is created from buffer. If the MemoryFlag.HostWriteOnly, MemoryFlag.HostReadOnly or MemoryFlag.HostNoAccess values are not specified in flags, they are inherited from the corresponding memory access qualifiers associated with buffer.
bufferCreateTypeDescribes the type of buffer object to be created.
bufferCreateInformationDescribes the type of buffer object to be created.
errorCodeReturns an appropriate error code. If errorCode is null, no error code is returned.
Returns
Returns a valid non-zero buffer object and errorCode is set to Result.Success if the buffer object is created successfully. Otherwise, it returns a null value and an error value in errorCode.
